# H3C_Switch_Log_Analyze

**基于 AI 的自动化网络巡检系统**

该系统专为 H3C 网络设备环境设计，通过 **Loki** 收集设备日志，利用 **大语言模型 (LLM)** 进行全局故障关联分析，并通过灵活的 **动态通道** 推送巡检报告。

---

## 🌟 核心特性

* **动态变量注入**：支持在配置文件中定义变量（如 `CORE_HOSTS`），并在 LogQL 查询语句中通过 `{VARIABLE}` 形式自动替换。
* **AI 关联分析**：不只是简单的日志堆砌。系统将多台设备的异常日志汇总给 AI（如 GLM-4, GPT-4），分析是否存在环路、级联故障或突发业务拥塞。
* **解耦的发送通道**：`send/` 文件夹下的模块支持动态加载，无需修改主程序即可切换或新增推送方式（如 Server酱、企业微信、钉钉）。
* **断点追溯逻辑**：记录上一次运行时间戳。如果由于 Cron 间隙（如深夜 0 点到 8 点）未运行，下次启动时会自动追溯这段时间内的所有日志。
* **故障隔离机制**：严格区分“Loki 查询失败”与“网络运行平稳（无告警）”，确保在数据源出问题时能及时收到系统告警。

## 📂 目录结构

```text
aiops/
├── check.py                # 主执行程序
├── README.md               # 项目说明文档
├── conf/                   # 配置文件目录
│   └── example.conf        # 巡检任务配置（支持多副本，对应不同任务）
├── send/                   # 发送插件目录
│   ├── __init__.py         # 必须存在（定义为 Python 包）
│   ├── serverchan.py       # 推送逻辑实现
│   └── serverchan.env      # 推送渠道密钥（单独管理，不进入主配置）
└── last_run.txt            # [自动生成] 记录上次运行时间（位于 /tmp 目录下）

```

## 🛠️ 环境准备

1. **Python 环境**：Python 3.8+
2. **依赖库安装**：
```bash
pip install requests

```


3. **Loki 与 Promtail**：确保您的 H3C 交换机日志已通过 Syslog 转发至 Loki，且 Label 包含 `job="switch"` 和 `host="[IP地址]"`。

---

## ⚙️ 配置指南

### 1. 任务配置 (conf/*.conf)

配置文件支持 `"""` 多行定义（用于 Prompt）。关键变量：

* `CORE_HOSTS`: 定义核心/汇聚交换机 IP 列表，用 `|` 分隔。
* `CORE_LOGQL`: 针对核心设备的查询语句，支持变量替换。
* `AI_PROMPT_TEMPLATE`: 发送给 AI 的指令。必须包含 `{combined_context}` 占位符。

### 2. 通道配置 (send/*.env)

例如使用 Server酱时，在 `send/serverchan.env` 中配置：
`SENDKEY=SCTxxxxxxxxxxx`

## 🚀 运行说明

### 1. 手动调试

执行脚本时必须传入配置文件的路径：

```bash
python3 check.py conf/example.conf

```

### 2. 自动化运维 (Cron)

为了配合脚本的追溯逻辑，建议配置以下 Cron 任务（每天 0 点及 8-23 点每小时整点运行）：

```bash
# 编辑 crontab -e
0 0,8-23 * * * cd /your/path/aiops && /usr/bin/python3 check.py conf/example.conf >> /tmp/aiops_cron.log 2>&1

```

---

## 💡 运维逻辑说明

* **状态保存**：`last_run.txt` 默认保存在 `/tmp` 目录下。
* **重启处理**：如果系统重启导致 `/tmp` 文件丢失，脚本在下次运行时会自动获取 **过去 1 小时** 的日志作为兜底。
* **运行间隔保护**：若两次运行间隔小于 1 小时，脚本将强制拉取 1 小时的深度，防止因频繁运行导致数据窗口过窄。
* **日志截断**：为防止单台设备产生海量重复日志（如接口反复震荡）导致 AI Token 超限，脚本会自动对设备日志进行首尾截断采样。

---

## ⚠️ 注意事项

* **路径权限**：请确保执行 Cron 的用户对 `check.py` 所在目录有读取权限，对 `/tmp` 有写入权限。
* **包定义**：如果 `send/` 目录下缺失 `__init__.py`，脚本将无法动态加载发送模块。
